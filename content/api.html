Belajar API

Pahami cara aplikasi berkomunikasi: REST, HTTP, Authentication, dan cara mengonsumsi API di frontend.

1

Apa itu API?

API (Application Programming Interface) adalah set aturan yang memungkinkan dua aplikasi berkomunikasi. Dalam konteks web, API memungkinkan frontend mengambil dan mengirim data ke backend.

Analogi Restoran:
- Kamu = Client (Frontend)
- Waiter = API (Menerima pesanan, membawa makanan)
- Dapur = Server/Backend (Memproses pesanan)
- Menu = API Documentation (Apa yang bisa dipesan) 
CLIENT (Frontend)              API               SERVER (Backend)
─────────────────          ─────────────          ────────────────
       │                       │                        │
       │── Request ────────────►                        │
       │   "GET /users"       │                        │
       │                       │──── Process ──────────►│
       │                       │                        │
       │                       │◄─── Data ─────────────│
       │◄── Response ──────────│                        │
       │   [{name: "John"}]    │                        │
Jenis API:
- REST API — Paling umum, menggunakan HTTP methods
- GraphQL — Query language, fleksibel
- SOAP — Legacy, enterprise
- WebSockets — Real-time communication
2

REST API Architecture

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun web services. RESTful API mengikuti prinsip-prinsip tertentu.

Prinsip REST:
1. Stateless — Setiap request independen, server tidak menyimpan state client
2. Client-Server — Separation of concerns
3. Cacheable — Response bisa di-cache
4. Uniform Interface — Consistent naming convention
5. Resource-based — Semua adalah resource (users, posts, etc)
RESTful Endpoints
# Contoh REST API untuk resource "users"

# Base URL
https://api.example.com/v1

# Endpoints
GET    /users           # List semua users
GET    /users/:id       # Detail user tertentu
POST   /users           # Buat user baru
PUT    /users/:id       # Update seluruh data user
PATCH  /users/:id       # Update sebagian data user
DELETE /users/:id       # Hapus user

# Nested resources
GET    /users/:id/posts      # Posts milik user
GET    /users/:id/posts/:postId
Naming Convention:
- Gunakan noun plural: /users, bukan /user
- Hindari verb: /getUsers (SALAH), /users (BENAR)
- Gunakan query untuk filter: /users?status=active
3

HTTP Methods (CRUD Operations)

HTTP methods mendefinisikan aksi yang ingin dilakukan pada resource. Setiap method punya makna spesifik dalam REST API.

HTTP Methods
GET — Read (Mengambil data)
├── Safe: Ya (tidak mengubah data)
├── Idempotent: Ya
└── Contoh: GET /users/123

POST — Create (Membuat data baru)
├── Safe: Tidak
├── Idempotent: Tidak (POST 2x = 2 data)
└── Contoh: POST /users { name: "John" }

PUT — Update (Replace seluruh data)
├── Safe: Tidak
├── Idempotent: Ya (PUT 10x = hasil sama)
└── Contoh: PUT /users/123 { name: "John", email: "..." }

PATCH — Partial Update (Update sebagian)
├── Safe: Tidak
├── Idempotent: Ya (tergantung implementasi)
└── Contoh: PATCH /users/123 { name: "New Name" }

DELETE — Delete (Menghapus data)
├── Safe: Tidak
├── Idempotent: Ya (DELETE data yang sudah terhapus = sama)
└── Contoh: DELETE /users/123
Idempotent: Operasi yang menghasilkan hasil sama meski dijalankan berkali-kali. GET, PUT, DELETE idempotent. POST tidak idempotent.
4

HTTP Status Codes

Status code adalah 3-digit angka yang menunjukkan hasil dari HTTP request. Memahami status code penting untuk debugging dan error handling.

Status Code Categories
/* 1xx - Informational */
100 Continue       // Request diterima, lanjutkan
101 Switching Protocols

/* 2xx - Success */
200 OK             // Request berhasil
201 Created        // Resource berhasil dibuat
204 No Content     // Berhasil tapi tidak ada body (untuk DELETE)

/* 3xx - Redirection */
301 Moved Permanently
302 Found (Redirect)
304 Not Modified   // Cache masih valid

/* 4xx - Client Error */
400 Bad Request    // Request tidak valid
401 Unauthorized   // Butuh autentikasi
403 Forbidden      // Tidak punya akses
404 Not Found      // Resource tidak ada
409 Conflict       // Konflik (misal: email sudah ada)
422 Unprocessable  // Validasi gagal
429 Too Many Requests // Rate limit exceeded

/* 5xx - Server Error */
500 Internal Server Error  // Error di server
502 Bad Gateway
503 Service Unavailable    // Server maintenance

Fetch API: GET Request

Fetch API adalah cara modern untuk melakukan HTTP request di JavaScript. Berikut cara mengambil data dari API.

JavaScript
// Basic GET request
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

// Dengan async/await (recommended)
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    
    // Cek apakah request berhasil
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Fetch error:', error);
  }
}

// Dengan query parameters
const params = new URLSearchParams({
  page: '1',
  limit: '10',
  sort: 'desc'
});

const response = await fetch(`https://api.example.com/users?${params}`);

// Dengan headers
const response = await fetch('https://api.example.com/users', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Accept': 'application/json'
  }
});

Fetch API: POST, PUT, DELETE

Untuk mengirim data ke server, gunakan method POST, PUT, atau DELETE dengan body berisi data yang akan dikirim.

JavaScript
// POST - Membuat data baru
async function createUser(userData) {
  const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify(userData)
  });
  
  return response.json();
}

// PUT - Update seluruh data
async function updateUser(id, userData) {
  const response = await fetch(`https://api.example.com/users/${id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(userData)
  });
  
  return response.json();
}

// DELETE - Hapus data
async function deleteUser(id) {
  const response = await fetch(`https://api.example.com/users/${id}`, {
    method: 'DELETE',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  
  if (response.status === 204) {
    console.log('User deleted successfully');
  }
}

// Penggunaan
const newUser = await createUser({
  name: 'John Doe',
  email: 'john@example.com'
});

API Authentication

Banyak API memerlukan authentication untuk mengakses protected endpoints. Berikut metode autentikasi yang umum digunakan.

Auth Methods
// 1. API Key (Simple)
fetch('https://api.example.com/data', {
  headers: {
    'X-API-Key': 'your-api-key-here'
  }
});

// atau sebagai query parameter
fetch('https://api.example.com/data?api_key=your-api-key');

// 2. Bearer Token (JWT)
fetch('https://api.example.com/users', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

// 3. Basic Auth (Not recommended for production)
const credentials = btoa(`${username}:${password}`);
fetch('https://api.example.com/data', {
  headers: {
    'Authorization': `Basic ${credentials}`
  }
});

// 4. OAuth 2.0 Flow (simplified)
// Redirect user to auth provider
const authUrl = `https://auth.provider.com/authorize?
  client_id=${clientId}&
  redirect_uri=${redirectUri}&
  response_type=code&
  scope=read write`;

// Exchange code for token di backend
Best Practice:
- Jangan simpan token di localStorage (vulnerable to XSS)
- Gunakan HttpOnly cookies untuk menyimpan token
- Implement refresh token mechanism
- Selalu gunakan HTTPS

Error Handling API

Error handling yang baik membantu debugging dan memberikan feedback yang jelas ke user.

JavaScript
class APIError extends Error {
  constructor(message, status, data) {
    super(message);
    this.status = status;
    this.data = data;
    this.name = 'APIError';
  }
}

async function apiRequest(url, options = {}) {
  try {
    const response = await fetch(url, options);
    
    if (!response.ok) {
      let errorData;
      try {
        errorData = await response.json();
      } catch {
        errorData = { message: response.statusText };
      }
      
      throw new APIError(
        errorData.message || 'Request failed',
        response.status,
        errorData
      );
    }
    
    return await response.json();
    
  } catch (error) {
    if (error instanceof APIError) {
      // Handle API errors
      switch (error.status) {
        case 401:
          console.error('Unauthorized - Redirect to login');
          break;
        case 403:
          console.error('Forbidden - No access');
          break;
        case 404:
          console.error('Not found');
          break;
        case 500:
          console.error('Server error');
          break;
        default:
          console.error(`Error: ${error.message}`);
      }
    } else {
      // Network error or other
      console.error('Network error:', error);
    }
    
    throw error;
  }
}

CORS (Cross-Origin Resource Sharing)

CORS adalah mekanisme keamanan browser yang membatasi web page mengakses resource dari domain berbeda.

Mengapa CORS Error?
Browser memblokir request ke domain berbeda demi keamanan. Server harus secara eksplisit mengizinkan request dari origin tertentu.
Server-Side CORS Headers
// Response headers yang harus di-set di server
Access-Control-Allow-Origin: https://yourdomain.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Credentials: true

// Express.js CORS setup
const cors = require('cors');

const corsOptions = {
  origin: ['https://yourdomain.com', 'http://localhost:3000'],
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  credentials: true
};

app.use(cors(corsOptions));

// Atau untuk development (JANGAN di production!)
app.use(cors({ origin: '*' }));
Solusi CORS Error:
1. Setup CORS headers di server
2. Gunakan proxy server untuk development
3. Pastikan origin (domain + port + protocol) sama persis

Axios Library

Axios adalah library populer untuk HTTP request dengan fitur lebih lengkap dibanding fetch.

JavaScript
// Install: npm install axios
import axios from 'axios';

// Create instance dengan config
const api = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' }
});

// Request interceptor (tambah token otomatis)
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor (handle errors globally)
api.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response.status === 401) {
      // Redirect to login
    }
    return Promise.reject(error);
  }
);

// Penggunaan
const users = await api.get('/users');
const user = await api.post('/users', { name: 'John' });
await api.delete(`/users/${id}`);

Rate Limiting & Pagination

API biasanya membatasi jumlah request per waktu tertentu. Pagination digunakan untuk data besar.

API Response
// Response headers untuk rate limiting
X-RateLimit-Limit: 100      // Max requests per hour
X-RateLimit-Remaining: 95   // Requests remaining
X-RateLimit-Reset: 1609459200  // Unix timestamp reset

// Pagination response
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8,
    "hasNext": true,
    "hasPrev": false
  },
  "links": {
    "first": "/users?page=1",
    "prev": null,
    "next": "/users?page=2",
    "last": "/users?page=8"

// Request dengan pagination
GET /users?page=2&limit=20&sort=-createdAt

GraphQL Introduction

GraphQL adalah query language untuk API yang memungkinkan client meminta exact data yang dibutuhkan.

GraphQL
// Query - Fetch data
query {
  user(id: "123") {
    name
    email
    posts {
      title
      comments {
        text
      }
    }
  }
}

// Mutation - Create/Update/Delete
mutation {
  createUser(input: {
    name: "John"
    email: "john@example.com"
  }) {
    id
    name
  }
}

// REST vs GraphQL
// REST:   GET /users/123
//         GET /users/123/posts
//         GET /posts/456/comments
// GraphQL: Satu request untuk semua data di atas!

Praktik: Konsumsi Public API

Gunakan public API untuk praktik. Contoh: JSONPlaceholder (fake API untuk testing).

JavaScript
// Contoh: Fetch posts dari JSONPlaceholder
async function fetchPosts() {
  try {
    const response = await fetch(
      'https://jsonplaceholder.typicode.com/posts'
    );
    
    if (!response.ok) {
      throw new Error(`HTTP error! ${response.status}`);
    }
    
    const posts = await response.json();
    
    posts.forEach(post => {
      console.log(`#${post.id}: ${post.title}`);
    });
    
  } catch (error) {
    console.error('Failed to fetch posts:', error);
  }
}

fetchPosts();

// Public APIs untuk praktik:
// - https://jsonplaceholder.typicode.com
// - https://rickandmortyapi.com
// - https://pokeapi.co
// - https://openweathermap.org/api
