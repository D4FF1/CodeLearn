Belajar Database

Kuasai penyimpanan data: SQL, NoSQL, database design, ORM, dan best practices mengelola data aplikasi.

1

Apa itu Database?

Database adalah sistem penyimpanan data yang terorganisir, memungkinkan aplikasi menyimpan, mengambil, dan mengelola informasi secara efisien.

Komponen Database:
1. DBMS (Database Management System) — Software untuk mengelola database
2. Schema — Struktur tabel dan relasi
3. Data — Informasi yang disimpan
4. Query Language — Bahasa untuk berinteraksi dengan data (SQL)

Mengapa Perlu Database?
- Persistensi data (tidak hilang saat restart)
- Query yang powerful
- Concurrency (banyak user akses bersamaan)
- Data integrity dan consistency
- Security dan access control
2

SQL vs NoSQL Database

Ada dua kategori utama database: SQL (Relational) dan NoSQL (Non-Relational). Pilih berdasarkan kebutuhan aplikasi.

SQL (Relational)

  • Structured data
  • Fixed schema
  • ACID compliant
  • Complex queries (JOIN)
  • Vertical scaling
  • PostgreSQL, MySQL, SQLite

NoSQL (Non-Relational)

  • Flexible schema
  • JSON-like documents
  • Horizontal scaling
  • High performance
  • Unstructured data
  • MongoDB, Redis, DynamoDB
Kapan Pilih Apa?
SQL: Aplikasi dengan data terstruktur, membutuhkan transaksi, relasi kompleks (e-commerce, banking, ERP)
NoSQL: Data tidak terstruktur, real-time analytics, content management, IoT, high write throughput

SQL Basics: SELECT, INSERT, UPDATE, DELETE

SQL (Structured Query Language) adalah bahasa standar untuk mengelola database relasional.

SQL
-- SELECT: Mengambil data
SELECT * FROM users;
SELECT name, email FROM users WHERE age > 18;
SELECT * FROM users 
WHERE city = 'Jakarta' 
ORDER BY created_at DESC
LIMIT 10 OFFSET 0;

-- INSERT: Menambah data
INSERT INTO users (name, email, age)
VALUES ('John Doe', 'john@email.com', 25);

-- Multiple insert
INSERT INTO users (name, email)
VALUES 
  ('Alice', 'alice@email.com'),
  ('Bob', 'bob@email.com');

-- UPDATE: Mengubah data
UPDATE users 
SET email = 'newemail@example.com', age = 26
WHERE id = 1;

-- DELETE: Menghapus data
DELETE FROM users WHERE id = 1;
DELETE FROM users WHERE last_login < '2023-01-01';
Warning: Selalu gunakan WHERE clause pada UPDATE dan DELETE. Tanpa WHERE, semua baris akan terpengaruh!

Membuat Table & Data Types

Table adalah struktur dasar penyimpanan di database relasional. Setiap kolom memiliki tipe data spesifik.

SQL
-- Membuat tabel users
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  age INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  balance DECIMAL(10,2) DEFAULT 0.00,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Data Types di PostgreSQL
-- INTEGER, BIGINT, SMALLINT  → Angka bulat
-- DECIMAL(precision, scale)   → Angka desimal (uang)
-- VARCHAR(n), TEXT            → Teks
-- BOOLEAN                     → true/false
-- DATE, TIME, TIMESTAMP       → Waktu
-- JSONB                       → JSON data
-- UUID                        → Unique identifier

-- Menambah kolom
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- Menghapus kolom
ALTER TABLE users DROP COLUMN phone;

-- Menghapus tabel
DROP TABLE IF EXISTS users;

Relationships: One-to-One, One-to-Many, Many-to-Many

Relasi antar tabel adalah inti dari database relasional. Memahami jenis relasi sangat penting untuk desain database yang baik.

SQL
-- ONE-TO-ONE: User punya satu profile
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);

CREATE TABLE profiles (
  id SERIAL PRIMARY KEY,
  user_id INTEGER UNIQUE REFERENCES users(id),
  bio TEXT,
  avatar_url VARCHAR(255)
);

-- ONE-TO-MANY: User punya banyak posts
CREATE TABLE posts (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),  -- Foreign key
  title VARCHAR(255),
  content TEXT
);

-- MANY-TO-MANY: Students <-> Courses
CREATE TABLE students (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);

CREATE TABLE courses (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100)
);

CREATE TABLE enrollments (  -- Junction table
  student_id INTEGER REFERENCES students(id),
  course_id INTEGER REFERENCES courses(id),
  enrolled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (student_id, course_id)
);

JOIN: Menggabungkan Tabel

JOIN digunakan untuk menggabungkan data dari dua atau lebih tabel berdasarkan kolom yang berhubungan.

SQL
-- INNER JOIN: Data yang ada di kedua tabel
SELECT users.name, posts.title
FROM users
INNER JOIN posts ON users.id = posts.user_id;

-- LEFT JOIN: Semua dari kiri, cocokan dari kanan
SELECT users.name, posts.title
FROM users
LEFT JOIN posts ON users.id = posts.user_id;
-- User tanpa post tetap muncul (posts.title = NULL)

-- RIGHT JOIN: Semua dari kanan
SELECT users.name, posts.title
FROM users
RIGHT JOIN posts ON users.id = posts.user_id;

-- FULL OUTER JOIN: Semua dari keduanya
SELECT users.name, posts.title
FROM users
FULL OUTER JOIN posts ON users.id = posts.user_id;

-- Multiple JOINs
SELECT 
  users.name,
  posts.title,
  comments.content
FROM users
JOIN posts ON users.id = posts.user_id
JOIN comments ON posts.id = comments.post_id;

-- Self JOIN: Tabel bergabung dengan dirinya sendiri
SELECT e1.name AS employee, e2.name AS manager
FROM employees e1
LEFT JOIN employees e2 ON e1.manager_id = e2.id;

Aggregation: GROUP BY, COUNT, SUM, AVG

Fungsi agregasi digunakan untuk menghitung nilai berdasarkan sekumpulan baris data.

SQL
-- COUNT: Menghitung jumlah baris
SELECT COUNT(*) AS total_users FROM users;
SELECT COUNT(email) AS users_with_email FROM users;
SELECT COUNT(DISTINCT city) AS unique_cities FROM users;

-- SUM, AVG, MIN, MAX
SELECT 
  SUM(balance) AS total_balance,
  AVG(balance) AS avg_balance,
  MIN(age) AS youngest,
  MAX(age) AS oldest
FROM users;

-- GROUP BY: Mengelompokkan data
SELECT city, COUNT(*) AS user_count
FROM users
GROUP BY city;

-- GROUP BY dengan multiple columns
SELECT city, is_active, COUNT(*) AS count
FROM users
GROUP BY city, is_active;

-- HAVING: Filter setelah GROUP BY (bukan WHERE!)
SELECT city, COUNT(*) AS user_count
FROM users
GROUP BY city
HAVING COUNT(*) > 10
ORDER BY user_count DESC;

-- Contoh kompleks: Stats per user
SELECT 
  users.name,
  COUNT(posts.id) AS post_count,
  SUM(posts.views) AS total_views,
  AVG(posts.views) AS avg_views
FROM users
LEFT JOIN posts ON users.id = posts.user_id
GROUP BY users.id, users.name
ORDER BY total_views DESC;

Indexes: Speed Up Queries

Index adalah struktur data yang mempercepat pencarian data. Tanpa index, database harus scan seluruh tabel (sequential scan).

SQL
-- Membuat index
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_city ON users(city);

-- Composite index (multiple columns)
CREATE INDEX idx_users_city_active ON users(city, is_active);

-- Unique index
CREATE UNIQUE INDEX idx_users_email_unique ON users(email);

-- Partial index (conditional)
CREATE INDEX idx_active_users ON users(email)
WHERE is_active = TRUE;

-- Lihat indexes
\di  -- PostgreSQL

-- Hapus index
DROP INDEX idx_users_city;

-- Check query execution plan
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@email.com';
-- Index Scan vs Sequential Scan
Kapan Perlu Index?
- Kolom yang sering di-WHERE clause
- Kolom yang sering di-JOIN
- Kolom yang sering di-ORDER BY

Trade-offs:
- Index memperlambat INSERT/UPDATE/DELETE
- Index menggunakan storage tambahan
- Jangan index semua kolom!

Transactions & ACID

Transaction adalah grup operasi yang dijalankan sebagai satu unit. ACID properties menjamin keandalan data.

ACID Properties:
- Atomicity — Semua operasi berhasil, atau tidak sama sekali
- Consistency — Data selalu dalam state valid
- Isolation — Transaksi paralel tidak saling mengganggu
- Durability — Data yang di-commit permanen tersimpan
SQL
-- Contoh: Transfer uang antar rekening
BEGIN;  -- Start transaction

-- Kurangi saldo pengirim
UPDATE accounts SET balance = balance - 100
WHERE id = 1;

-- Tambah saldo penerima
UPDATE accounts SET balance = balance + 100
WHERE id = 2;

-- Commit jika semua berhasil
COMMIT;

-- Atau ROLLBACK jika ada error
-- ROLLBACK;

-- Dengan SAVEPOINT
BEGIN;
INSERT INTO orders (user_id) VALUES (1);
SAVEPOINT order_created;

INSERT INTO order_items VALUES (1, 'Product A');
-- Jika error, bisa rollback ke savepoint
ROLLBACK TO order_created;

COMMIT;

Connecting Node.js to PostgreSQL

Menghubungkan aplikasi Node.js dengan PostgreSQL menggunakan pg (node-postgres) driver.

JavaScript
// Install: npm install pg
const { Pool } = require('pg');

// Connection pool (recommended)
const pool = new Pool({
  host: process.env.DB_HOST,
  port: process.env.DB_PORT || 5432,
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  max: 20,  // Max connections in pool
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000
});

// Simple query
const result = await pool.query('SELECT * FROM users WHERE id = $1', [1]);
console.log(result.rows);

// Parameterized query (PREVENT SQL INJECTION!)
const text = 'INSERT INTO users(name, email) VALUES($1, $2) RETURNING *';
const values = ['John', 'john@email.com'];
const res = await pool.query(text, values);

// Transaction
const client = await pool.connect();
try {
  await client.query('BEGIN');
  
  await client.query('UPDATE accounts SET balance = balance - $1 WHERE id = $2', [100, 1]);
  await client.query('UPDATE accounts SET balance = balance + $1 WHERE id = $2', [100, 2]);
  
  await client.query('COMMIT');
} catch (err) {
  await client.query('ROLLBACK');
  throw err;
} finally {
  client.release();
}

ORM: Prisma

ORM (Object-Relational Mapping) memungkinkan berinteraksi dengan database menggunakan code, bukan raw SQL. Prisma adalah ORM modern untuk Node.js.

JavaScript
// Install: npm install prisma --save-dev
// npx prisma init

// prisma/schema.prisma
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String?
  posts     Post[]
  createdAt DateTime @default(now())
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String?
  author    User     @relation(fields: [authorId], references: [id])
  authorId  Int
}

// Query dengan Prisma Client
const { PrismaClient } = require('@prisma/client');
const prisma = new PrismaClient();

// Create
const user = await prisma.user.create({
  data: {
    name: 'John',
    email: 'john@email.com'
  }
});

// Read
const users = await prisma.user.findMany({
  where: { name: { contains: 'John' } },
  include: { posts: true },
  orderBy: { createdAt: 'desc' }
});

// Update
await prisma.user.update({
  where: { id: 1 },
  data: { name: 'Updated Name' }
});

// Delete
await prisma.user.delete({ where: { id: 1 } });
Kelebihan Prisma:
- Type-safe dengan TypeScript
- Auto-generated client
- Migration system built-in
- Intellisense di VS Code
- Visual database browser (Prisma Studio)

MongoDB: NoSQL Database

MongoDB adalah database NoSQL document-based. Data disimpan dalam format BSON (binary JSON), fleksibel tanpa fixed schema.

JavaScript
// Install: npm install mongoose
const mongoose = require('mongoose');

// Connect
await mongoose.connect(process.env.MONGODB_URI);

// Define Schema
const userSchema = new mongoose.Schema({
  name: { type: String, required: true },
  email: { type: String, required: true, unique: true },
  age: { type: Number, min: 0 },
  tags: [String],
  address: {
    street: String,
    city: String,
    country: String
  }
}, { timestamps: true });

const User = mongoose.model('User', userSchema);

// CRUD Operations
// Create
const user = await User.create({
  name: 'John Doe',
  email: 'john@email.com',
  tags: ['developer', 'designer']
});

// Read
const users = await User.find({ age: { $gte: 18 } })
  .sort({ createdAt: -1 })
  .limit(10);

// Update
await User.findByIdAndUpdate(id, { name: 'New Name' }, { new: true });

// Delete
await User.findByIdAndDelete(id);

Database Migrations

Migration adalah cara version control untuk database. Setiap perubahan schema dicatat sebagai file yang bisa dijalankan ulang.

Terminal
# Prisma Migrations
npx prisma migrate dev --name init
npx prisma migrate dev --name add_user_avatar
npx prisma migrate deploy  # Production

# Knex.js Migrations
npx knex migrate:make create_users_table
npx knex migrate:latest
npx knex migrate:rollback

# Sequelize Migrations
npx sequelize-cli migration:generate --name create-users
npx sequelize-cli db:migrate
npx sequelize-cli db:migrate:undo:all
JavaScript (Knex Migration)
// migrations/20240101000000_create_users.js
exports.up = function(knex) {

    table.increments('id').primary();
    table.string('name', 100).notNullable();

  });

exports.down = function(knex) {
  return knex.schema.dropTableIfExists('users');
};

Backup & Restore

Backup sangat penting untuk recovery data. Gunakan tools yang disediakan DBMS.

Terminal
# PostgreSQL Backup
# Full database backup
pg_dump -U postgres -d mydb -F c -f backup.dump

# SQL format (human readable)
pg_dump -U postgres -d mydb > backup.sql

# Backup specific tables
pg_dump -U postgres -d mydb -t users -t posts > tables_backup.sql

# Restore
pg_restore -U postgres -d mydb backup.dump
psql -U postgres -d mydb -f backup.sql

# MongoDB Backup
mongodump --uri="mongodb://localhost:27017/mydb" --out=./backup

# MongoDB Restore
mongorestore --uri="mongodb://localhost:27017/mydb" ./backup/mydb

# MySQL Backup
mysqldump -u root -p mydb > backup.sql

# MySQL Restore
mysql -u root -p mydb < backup.sql

Performance Optimization

Optimasi performa database sangat penting untuk aplikasi dengan traffic tinggi.

SQL
-- ANALYZE query execution
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@email.com';

-- Avoid SELECT *
-- Bad
SELECT * FROM users;

-- Good
SELECT id, name, email FROM users;

-- Use LIMIT for pagination
SELECT * FROM posts ORDER BY created_at DESC LIMIT 20 OFFSET 0;

-- Use EXISTS instead of IN for subqueries
SELECT * FROM users u
WHERE EXISTS (SELECT 1 FROM posts p WHERE p.user_id = u.id);

-- Batch inserts
INSERT INTO users (name, email) VALUES
  ('User 1', 'user1@email.com'),
  ('User 2', 'user2@email.com'),
  ('User 3', 'user3@email.com');

-- Connection pooling configuration
-- Di aplikasi: gunakan pool, bukan single connection
Tips Optimasi:
- Index kolom yang sering di-query
- Hindari N+1 query problem
- Gunakan connection pooling
- Monitor slow queries
- Denormalize jika perlu (untuk read-heavy)

Database Security

Keamanan database mencakup akses control, encryption, dan perlindungan dari serangan.

SQL
-- Create limited user (principle of least privilege)
CREATE USER app_user WITH PASSWORD 'secure_password';

-- Grant specific permissions only
GRANT SELECT, INSERT, UPDATE, DELETE ON users TO app_user;
GRANT USAGE, SELECT ON SEQUENCE users_id_seq TO app_user;

-- Revoke unnecessary permissions
REVOKE ALL ON users FROM PUBLIC;

-- Row Level Security (PostgreSQL)
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
CREATE POLICY user_posts_policy ON posts
  USING (user_id = current_user_id());

-- Parameterized queries prevent SQL injection
-- BAD (SQL Injection vulnerable!)
const query = `SELECT * FROM users WHERE id = ${userId}`;

-- GOOD (Parameterized)
const query = 'SELECT * FROM users WHERE id = $1';
const result = await pool.query(query, [userId]);
Security Checklist:
- Jangan root/superuser di aplikasi
- Encrypt sensitive data (passwords, credit cards)
- Gunakan SSL untuk connection
- Backup encryption
- Regular security updates

Praktik: Database Design untuk E-commerce

Desain database sederhana untuk aplikasi e-commerce dengan users, products, orders.

SQL
-- Users table
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Products table
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  stock INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Orders table
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  total DECIMAL(10,2) NOT NULL,
  status VARCHAR(50) DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Order items (many-to-many: orders <-> products)
CREATE TABLE order_items (
  id SERIAL PRIMARY KEY,
  order_id INTEGER REFERENCES orders(id) ON DELETE CASCADE,
  product_id INTEGER REFERENCES products(id),
  quantity INTEGER NOT NULL,
  price DECIMAL(10,2) NOT NULL
);

-- Create indexes for performance
CREATE INDEX idx_orders_user ON orders(user_id);
CREATE INDEX idx_order_items_order ON order_items(order_id);
CREATE INDEX idx_products_price ON products(price);
