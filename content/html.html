25 Lessons + Penjelasan

Belajar HTML

Pelajari fondasi web: sintaks HTML, semantic elements, forms, media, dan accessibility. Setiap materi dilengkapi penjelasan mendalam.

1

Pengenalan & Dasar Dokumen HTML

Setiap halaman web dimulai dari struktur HTML. DOCTYPE adalah deklarasi yang memberitahu browser bahwa dokumen ini menggunakan HTML versi terbaru (HTML5). Tag-tag seperti <html>, <head>, dan <body> membentuk kerangka dasar yang harus dipahami dengan benar.

Penjelasan Detail: Dokumen HTML selalu diawali dengan <!DOCTYPE html> agar browser tidak masuk ke mode aneh (quirks mode). Elemen <html> adalah akar, sedangkan <head> berisi metadata (judul, charset, viewport) yang tidak terlihat. <body> adalah wadah semua konten yang tampil. Pastikan selalu menyertakan meta charset="UTF-8" untuk mendukung huruf Indonesia.
HTML 
<!DOCTYPE html>
<html lang="id">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Judul Halaman Web Saya</title>
  </head>
  <body>
    <h1>Selamat Datang di Website Saya!</h1>
    <p>Ini adalah paragraf pertama saya.</p>
  </body>
</html>
Penjelasan Setiap Elemen:
  • <!DOCTYPE html> — WAJIB di baris paling atas.
  • <html lang="id"> — Elemen root, atribut lang penting untuk aksesibilitas.
  • <head> — Metadata, tidak tampil.
  • <meta charset="UTF-8"> — Mendukung karakter unik.
  • <meta name="viewport"> — Responsif di HP.
  • <title> — Judul di tab browser.
  • <body> — Area konten yang terlihat.
2

Headings (h1-h6), Paragraf, HR, BR

Heading Tags <h1> sampai <h6> menunjukkan tingkatan hierarki dokumen. H1 adalah yang paling penting, biasanya judul utama halaman.

Penjelasan: Heading bukan hanya untuk memperbesar teks, tetapi untuk memberikan struktur logis. Google dan pembaca layar menggunakan heading untuk memahami alur konten. Hanya boleh ada satu <h1> per halaman. Gunakan <h2> untuk sub-bab, <h3> untuk sub-sub, dst. Jangan melompat (misal dari H2 langsung ke H4).
HTML
<h1>Judul Utama</h1>
<h2>Sub Bagian</h2>
<p>Paragraf dengan <strong>tebal</strong> dan <em>miring</em>.</p>
<hr>
<p>Garis pemisah di atas.<br>Baris baru.</p>
Kesalahan Umum: Memakai heading berdasarkan ukuran, bukan tujuan. Jangan gunakan <br> beruntun untuk memberi jarak.
3

Struktur & Elemen Semantik HTML5

Elemen semantik adalah tags yang menjelaskan MAKSUD dari konten, bukan hanya tampilannya.

Penjelasan: <header>, <nav>, <main>, <article>, <aside>, <footer> memberikan makna struktural. <main> hanya boleh satu per halaman. <article> untuk konten yang bisa berdiri sendiri. Ini meningkatkan SEO dan aksesibilitas.
HTML
<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    <article>...</article>
    <aside>...</aside>
  </main>
  <footer>...</footer>
</body>
4

Komentar & Organisasi Kode

Komentar dalam HTML adalah teks yang tidak ditampilkan browser, berguna untuk dokumentasi.

Penjelasan: Gunakan <!-- ... --> untuk memberi catatan pada kode. Komentar dapat digunakan untuk menonaktifkan sementara bagian kode. Pastikan tidak meninggalkan komentar yang berisi informasi sensitif.
HTML
<!-- Ini komentar, tidak tampil di browser -->
<!-- TODO: perbaiki navigasi -->
<div>Konten aktif</div>
5

Links (a, href, target, rel)

Element <a> (anchor) menghubungkan halaman ke halaman lain.

Penjelasan: Atribut href adalah tujuan. Gunakan target="_blank" untuk membuka di tab baru, selalu sertakan rel="noopener noreferrer" untuk keamanan. mailto: dan tel: untuk email dan telepon.
HTML
<a href="https://google.com" target="_blank" rel="noopener">Google</a>
<a href="#footer">Ke footer</a>
<a href="mailto:admin@example.com">Email</a>
6

Images (img, src, alt, figure/figcaption)

Element <img> menampilkan gambar. Atribut alt WAJIB untuk aksesibilitas.

Penjelasan: src adalah sumber gambar. alt memberikan deskripsi untuk screen reader. Gunakan loading="lazy" untuk performa. <figure> dan <figcaption> untuk mengelompokkan gambar dengan keterangan.
HTML
<img src="foto.jpg" alt="Deskripsi gambar" loading="lazy">
<figure>
  <img src="chart.png" alt="Grafik penjualan">
  <figcaption>Grafik 2024</figcaption>
</figure>
7

Lists (ul/ol/li, Description Lists dl/dt/dd)

List adalah cara menampilkan kumpulan item. Ada unordered (bullet), ordered (nomor), dan description list.

Penjelasan: <ul> untuk daftar tanpa urutan, <ol> untuk daftar berurutan (bisa diubah type-nya: 1, A, a, I). <dl> digunakan untuk pasangan istilah-definisi.
HTML
<ul>
  <li>Apel</li>
  <li>Pisang</li>
</ul>

<ol type="A">
  <li>Langkah 1</li>
  <li>Langkah 2</li>
</ol>

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>
8

Tables (table, thead/tbody/tfoot, colspan/rowspan)

Tabel untuk menampilkan data tabular secara terstruktur.

Penjelasan: Jangan gunakan tabel untuk layout halaman. Gunakan <th> untuk header, colspan dan rowspan untuk menggabungkan sel. Atribut scope pada th membantu aksesibilitas.
HTML
<table>
  <thead>
    <tr>
      <th>Nama</th>
      <th>Skor</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Andi</td>
      <td>90</td>
    </tr>
  </tbody>
</table>
9

Forms (form, label, input, textarea, select)

Form adalah mekanisme utama interaksi user dengan website.

Penjelasan: Setiap input harus memiliki <label> yang terhubung dengan atribut for. Atribut name penting karena menjadi nama data saat dikirim ke server. Gunakan method="POST" untuk data sensitif.
HTML
<form action="/proses" method="post">
  <label for="nama">Nama:</label>
  <input type="text" id="nama" name="nama" required>
  <button type="submit">Kirim</button>
</form>
10

Input Types Lengkap

Penjelasan: Memilih tipe input yang tepat meningkatkan UX: email menampilkan keyboard dengan @, tel memunculkan numeric keypad di HP, number memiliki tombol naik/turun, date menampilkan kalender. checkbox untuk pilihan banyak, radio untuk pilihan tunggal.
HTML
<input type="email" placeholder="email@contoh.com">
<input type="number" min="1" max="100">
<input type="date">
<input type="checkbox" id="setuju">
<input type="radio" name="gender" value="L">
<input type="file" accept="image/*">
11

Atribut Universal (id, class, style, data-*)

Penjelasan: id harus unik dalam satu halaman. class bisa digunakan banyak elemen. data-* untuk menyimpan data custom yang bisa diakses via JavaScript (element.dataset). hidden menyembunyikan elemen.
HTML
<div id="main" class="box" data-user-id="123">Konten</div>
<p contenteditable="true">Bisa diedit</p>
12

Form Validation (required, pattern, minlength)

Penjelasan: HTML5 menyediakan validasi bawaan: required (tidak boleh kosong), minlength/maxlength, pattern (regex), min/max untuk number. Browser akan menampilkan pesan error dan mencegah submit.
HTML
<input type="text" required minlength="3">
<input type="email" required>
<input pattern="[A-Za-z]{3,}" title="Minimal 3 huruf">
13

Buttons (button, submit, reset)

Penjelasan: <button> lebih fleksibel daripada <input type="button"> karena bisa berisi HTML. type="submit" untuk mengirim form, type="reset" untuk mereset, type="button" untuk tombol biasa yang dikontrol JavaScript.
HTML
<button type="button">Klik Saya</button>
<button type="submit">Simpan</button>
<button type="reset">Reset Form</button>
14

Media (audio, video, source, poster)

Penjelasan: Gunakan <audio> dan <video> untuk menyematkan media. Atribut controls menampilkan panel play/pause. poster untuk thumbnail video. Sediakan multiple source untuk kompatibilitas browser.
HTML
<audio controls>
  <source src="lagu.mp3" type="audio/mpeg">
</audio>

<video controls width="400" poster="thumb.jpg">
  <source src="video.mp4" type="video/mp4">
</video>
15

Iframe & Security (sandbox)

Penjelasan: <iframe> menyematkan halaman lain. Atribut sandbox membatasi kemampuan iframe untuk keamanan. Selalu berikan title untuk aksesibilitas.
HTML
<iframe 
  src="https://example.com" 
  width="600" 
  height="400"
  title="Contoh iframe"
  sandbox="allow-same-origin allow-scripts">
</iframe>
16

Special Elements (div/span, strong/em, code/pre, kbd)

Penjelasan: <div> block-level untuk grouping, <span> inline untuk styling. <strong> untuk penting (bold), <em> untuk penekanan (miring). <code> untuk potongan kode, <pre> untuk teks praformat. <kbd> untuk input keyboard.
HTML
<p>Tekan <kbd>Ctrl+S</kbd> untuk menyimpan.</p>
<pre><code>function hello() { 
  console.log("Halo"); 
}</code></pre>
17

Layout Dasar (semantic wrappers)

Penjelasan: Gunakan elemen semantik untuk membuat kerangka layout: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>. Ini membantu SEO dan aksesibilitas.
HTML
<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    <section>...</section>
  </main>
  <footer>...</footer>
</body>
18

Accessibility (alt, label, heading order, landmarks)

Penjelasan: Aksesibilitas membuat web dapat digunakan oleh semua orang. Pastikan: semua gambar punya alt, semua input punya <label>, heading berurutan (h1 ke h2 ke h3), dan gunakan landmark (main, nav, dll).
HTML
<img src="icon.png" alt="Ikon pencarian">
<label for="email">Email:</label>
<input type="email" id="email">
<button aria-label="Tutup dialog">X</button>
19

Meta Tags (charset, description, og basic)

Penjelasan: Meta tag memberikan informasi ke browser dan mesin pencari. description digunakan oleh Google untuk snippet hasil pencarian. Open Graph (og:) mengontrol tampilan saat link dibagikan di media sosial.
HTML
<meta name="description" content="Belajar HTML dari dasar">
<meta property="og:title" content="Judul Website">
<meta property="og:image" content="https://domain/preview.jpg">
20

HTML Character Entities

Penjelasan: Karakter seperti <, >, &, ", © memiliki arti khusus di HTML. Gunakan entity: &lt; untuk <, &gt; untuk >, &amp; untuk &, &copy; untuk ©. &nbsp; adalah spasi non-breaking.
HTML
&lt; = <
&gt; = >
&amp; = &
&copy; = ©
&nbsp; = spasi non-breaking
21

Path Relative vs Absolute (href, src)

Penjelasan: Absolute path mencakup domain lengkap (https://example.com/gambar.jpg). Relative path berdasarkan lokasi file saat ini: gambar.jpg (satu folder), images/gambar.jpg (subfolder), ../gambar.jpg (folder parent). Root-relative dimulai dengan slash: /assets/logo.png.
HTML
<!-- Absolute -->
<img src="https://domain.com/foto.jpg">

<!-- Relative -->
<img src="foto.jpg">
<img src="../assets/icon.png">

<!-- Root-relative -->
<a href="/produk/sepatu.html">Produk</a>
22

Custom Data Attribute (data-*)

Penjelasan: Atribut data-* menyimpan data kustom pada elemen HTML. Contoh: data-product-id="123". Di JavaScript dapat diakses via element.dataset.productId. Berguna untuk menyimpan data sementara.
HTML
<div class="product" data-id="123" data-price="50000">
  Sepatu
</div>

<button data-action="delete" data-item-id="5">
  Hapus
</button>
23

Responsive Images Basics (srcset, sizes)

Penjelasan: srcset menyediakan beberapa versi gambar untuk lebar layar berbeda. sizes memberi tahu browser perkiraan lebar gambar pada berbagai breakpoint. Browser otomatis memilih gambar terbaik.
HTML
<img
  src="small.jpg"
  srcset="medium.jpg 800w, large.jpg 1200w"
  sizes="(max-width: 600px) 480px, 800px"
  alt="Pemandangan">
24

Progressive Enhancement (konsep)

Penjelasan: Progressive enhancement adalah strategi membangun website dengan konten inti (HTML) terlebih dahulu yang dapat diakses di semua perangkat, kemudian menambahkan lapisan CSS untuk tampilan, dan JS untuk interaktivitas. Ini memastikan website tetap berfungsi meskipun CSS/JS gagal dimuat.
HTML
<!-- Layer 1: konten (selalu work) -->
<a href="page.html">Link</a>

<!-- Layer 2: CSS (enhancement) -->
<!-- Layer 3: JavaScript (interaktivitas) -->
Prinsip: HTML dulu (struktur dan konten), kemudian CSS (presentasi), terakhir JavaScript (behavior). Website harus tetap usable tanpa CSS/JS.
25

Praktik: Bikin Landing Sederhana

Penjelasan: Gunakan semua elemen yang telah dipelajari: struktur semantik, heading, paragraf, gambar, list, form, dan link. Buatlah halaman landing sederhana dengan header, hero section, fitur, dan footer.
HTML
<!DOCTYPE html>
<html lang="id">
<head>
  <meta charset="UTF-8">
  <title>Landing Page Saya</title>
</head>
<body>
  <header>
    <h1>Produk Hebat</h1>
    <nav>
      <a href="#fitur">Fitur</a>
      <a href="#kontak">Kontak</a>
    </nav>
  </header>
  
  <main>
    <section>
      <h2>Solusi Terbaik untuk Anda</h2>
      <p>Bergabunglah sekarang dan nikmati manfaatnya.</p>
      <a href="#">Daftar Gratis</a>
    </section>
    
    <section id="fitur">
      <h2>Fitur Unggulan</h2>
      <ul>
        <li>Cepat dan Efisien</li>
        <li>Mudah Digunakan</li>
        <li>Aman dan Terpercaya</li>
      </ul>
    </section>
  </main>
  
  <footer>
    <p>&copy; 2025 Produk Hebat. All rights reserved.</p>
  </footer>
</body>
</html>
Ceklist Proyek:
  • DOCTYPE dan struktur dasar HTML
  • Semantic elements (header, main, footer, nav, section)
  • Heading yang terstruktur (h1, h2)
  • Links untuk navigasi
  • List untuk fitur
  • Character entity untuk copyright
