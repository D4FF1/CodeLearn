Belajar Framework Frontend

Pahami konsep modern: Component, State, Virtual DOM, dan memilih framework yang tepat untuk project kamu.

1

Mengapa Perlu Framework?

Framework frontend menyediakan struktur dan tools untuk membangun aplikasi web kompleks lebih efisien.

Masalah dengan Vanilla JS:
- Manipulasi DOM manual merepotkan
- Tidak ada struktur standar
- Sulit mengelola state di aplikasi besar
- Re-render tidak efisien
- Code reuseability terbatas

Tanpa Framework

  • Manual DOM manipulation
  • jQuery spaghetti code
  • Global variables everywhere
  • Sulit scale untuk tim besar

Dengan Framework

  • Declarative UI
  • Component-based architecture
  • State management built-in
  • Efficient updates (Virtual DOM)
Framework Populer 2024:
- React — Library UI dari Meta, ecosystem terbesar
- Vue — Progressive framework, mudah dipelajari
- Svelte — Compiler-based, no Virtual DOM
- Angular — Full framework dari Google, enterprise-ready
- Solid — Reactive without Virtual DOM, performa tinggi
2

Component-Based Architecture

Component adalah blok bangunan reusable yang menggabungkan HTML, CSS, dan JavaScript dalam satu unit independen.

Prinsip Component:
1. Encapsulation — Logic dan styling terisolasi
2. Reusability — Pakai berkali-kali di tempat berbeda
3. Composition — Gabung component kecil jadi besar
4. Single Responsibility — Satu tugas spesifik
React Component
// Button.jsx - Reusable component
function Button({ children, onClick, variant = 'primary' }) {
  return (
    <button
      className={`btn btn-${variant}`}
      onClick={onClick}
    >
      {children}
    </button>
  );
}

// Penggunaan
<Button variant="success" onClick={handleSubmit}>
  Simpan
</Button>
Vue Component
<!-- Button.vue -->
<template>
  <button :class="['btn', `btn-${variant}`]" @click="$emit('click')">
    <slot />
  </button>
</template>

<script setup>
defineProps({
  variant: { type: String, default: 'primary' }
});
</script>

JSX: JavaScript + XML

JSX adalah syntax extension untuk JavaScript yang memungkinkan menulis markup dalam JavaScript. JSX bukan HTML, tapi syntactic sugar untuk function calls.

JSX vs JavaScript
// JSX
const element = (
  <div className="card">
    <h1>Hello {name}</h1>
    <p>Welcome</p>
  </div>
);

// Compiled to JavaScript (what browser sees)
const element = React.createElement(
  'div',
  { className: 'card' },
  React.createElement('h1', null, 'Hello ', name),
  React.createElement('p', null, 'Welcome')
);

// Rules of JSX:
// 1. Single root element
// 2. Close all tags: <img />, <br />
// 3. className instead of class
// 4. JSX expressions in {}
// 5. Conditional: {condition && <Elem />} or {cond ? <A/> : <B/>}

Props: Data dari Parent ke Child

Props adalah cara component menerima data dari parent. Props bersifat read-only (immutable), child tidak boleh mengubahnya.

React Props
// Parent Component
function App() {
  return (
    <UserCard
      name="John Doe"
      age={25}
      isActive={true}
      hobbies={['coding', 'gaming']}
    />
  );
}

// Child Component (destructuring)
function UserCard({ name, age, isActive, hobbies }) {
  return (
    <div className="card">
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <span>Status: {isActive ? 'Active' : 'Inactive'}</span>
      <ul>
        {hobbies.map(hobby => <li key={hobby}>{hobby}</li>)}
      </ul>
    </div>
  );
}

// Default props
function Button({ text = 'Click me', color = 'blue' }) {
  return <button style={{ color }}>{text}</button>;
}
5

State: Data Internal Component

State adalah data internal yang bisa berubah dan memicu re-render saat diubah. Berbeda dengan props yang dari luar, state dikelola di dalam component.

React useState Hook
import { useState } from 'react';

function Counter() {
  // [currentState, setterFunction] = useState(initialValue)
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount(prevCount => prevCount + 1);
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>+</button>
    </div>
  );
}

// State dengan object
const [user, setUser] = useState({
  name: '',
  email: ''
});

// Update partial object (spread operator!)
setUser({ ...user, name: 'John' });

// State dengan array
const [items, setItems] = useState([]);

// Add item (immutably)
setItems([...items, newItem]);

// Remove item
setItems(items.filter(item => item.id !== id));
Aturan Penting:
- Jangan mutate state langsung (state.push = SALAH)
- State updates adalah async
- Gunakan functional update untuk nilai baru berdasarkan nilai lama
6

Virtual DOM & Reconciliation

Virtual DOM adalah representasi JavaScript dari DOM sebenarnya. Framework membuat perubahan di Virtual DOM dulu, lalu mengupdate DOM nyata secara efisien.

Proses Reconciliation:
1. State berubah → Virtual DOM baru dibuat
2. Compare Virtual DOM baru vs lama
3. Hitung minimal changes
4. Apply changes ke Real DOM

Ini membuat update sangat cepat karena manipulasi DOM (mahal) hanya dilakukan pada bagian yang benar-benar berubah.
Key Prop: Memberikan key pada list items membantu React mengidentifikasi item mana yang berubah, ditambah, atau dihapus. Gunakan ID unik, BUKAN index!
7

Lifecycle & useEffect Hook

Component memiliki lifecycle: mount (dibuat), update (re-render), dan unmount (dihapus). useEffect hook mengelola side effects di React.

React useEffect
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  // Run on mount and when userId changes
  useEffect(() => {
    async function fetchUser() {
      const response = await fetch(`/api/users/${userId}`);
      const data = await response.json();
      setUser(data);
    }
    fetchUser();

    // Cleanup function (run on unmount)
    return () => {
      console.log('Cleanup!');
    };
  }, [userId]); // Dependency array

  if (!user) return <p>Loading...</p>;

  return <h1>{user.name}</h1>;
}

// Dependency array variations:
useEffect(() => {});           // Run on EVERY render
useEffect(() => {}, []);        // Run ONCE on mount
useEffect(() => {}, [dep]);     // Run when dep changes
8

Event Handling

Framework menggunakan synthetic events yang cross-browser. Penanganan event mirip dengan vanilla JS tapi dengan sintaks sedikit berbeda.

React Events
function Form() {
  const [value, setValue] = useState('');

  // Event handler menerima synthetic event
  const handleChange = (e) => {
    setValue(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Submitted:', value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={value}
        onChange={handleChange}
      />
      <button type="submit">Send</button>
    </form>
  );
}

// Passing arguments to handlers
<button onClick={() => handleDelete(id)}>Delete</button>
Perbedaan dengan HTML:
- onClick bukan onclick
- Handler menerima function, bukan string
- preventDefault() untuk stop default action

Conditional Rendering

Menampilkan component atau elemen berdasarkan kondisi. Framework menggunakan JavaScript native untuk conditional rendering.

Conditional Patterns
// 1. If statement (tidak bisa di JSX)
if (isLoading) {
  return <Spinner />;
}
return <Content />;

// 2. Ternary operator
{isLoggedIn ? <Dashboard /> : <Login />}

// 3. Logical AND (&&)
{showModal && <Modal />}

// 4. Immediately invoked function
{(() => {
  switch (status) {
    case 'loading': return <Spinner />;
    case 'error': return <Error />;
    default: return <Content />;
  }
})()}

Rendering Lists & Keys

Menampilkan array data sebagai list elemen. Key prop sangat penting untuk performa dan kebenaran rendering.

List Rendering
const products = [
  { id: 1, name: 'Laptop', price: 999 },
  { id: 2, name: 'Phone', price: 699 },
  { id: 3, name: 'Tablet', price: 499 },
];

function ProductList() {
  return (
    <ul>
      {products.map(product => (
        <li key={product.id}>
          {product.name} - ${product.price}
        </li>
      ))}
    </ul>
  );
}

// Key harus UNIK di antara siblings
// BAIK: key={item.id}
// BURUK: key={index} (hanya jika tidak ada ID dan list static)

Controlled vs Uncontrolled Forms

Di React, ada dua cara menangani form: controlled (state-driven) dan uncontrolled (ref-based). Controlled lebih umum digunakan.

Controlled Form
function ContactForm() {
  const [form, setForm] = useState({
    name: '',
    email: '',
    message: ''
  });

  const handleChange = (e) => {
    setForm({
      ...form,
      [e.target.name]: e.target.value
    });
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Form submitted:', form);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="name"
        value={form.name}
        onChange={handleChange}
      />
      <textarea
        name="message"
        value={form.message}
        onChange={handleChange}
      />
      <button type="submit">Kirim</button>
    </form>
  );
}

State Management (Context, Redux, Zustand)

Untuk aplikasi kompleks, state perlu dibagikan antar component. Berbagai solusi tersedia dari built-in hingga library eksternal.

Pilihan State Management:
1. useState/useReducer — Local state, cocok untuk state simple
2. Context API — Share state tanpa prop drilling, built-in React
3. Redux — Predictable state container, cocok untuk app besar
4. Zustand — Lightweight alternative, simpler API
5. Jotai/Recoil — Atomic state management
Context API
// 1. Create Context
const UserContext = createContext();

// 2. Provider Component
function UserProvider({ children }) {
  const [user, setUser] = useState(null);
  
  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  );
}

// 3. Consume in any component
function Profile() {
  const { user } = useContext(UserContext);
  return <h1>Hello, {user?.name}</h1>;
}

Client-Side Routing

SPA (Single Page Application) menggunakan client-side routing untuk navigasi tanpa reload halaman penuh.

React Router
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/users/:id" element={<UserDetail />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}

// Dynamic route parameter
function UserDetail() {
  const { id } = useParams();
  return <h1>User ID: {id}</h1>;
}

Data Fetching Patterns

Mengambil data dari API adalah tugas umum. Ada berbagai pendekatan dari fetch native hingga library khusus.

Custom Hook for Fetching
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    async function fetchData() {
      try {
        const res = await fetch(url);
        const json = await res.json();
        setData(json);
      } catch (err) {
        setError(err);
      } finally {
        setLoading(false);
      }
    }
    fetchData();
  }, [url]);

  return { data, loading, error };
}

// Usage
function Users() {
  const { data, loading, error } = useFetch('/api/users');
  
  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error!</p>;
  return <ul>{data.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}
Alternatif Populer:
- TanStack Query (React Query) — Powerful data fetching & caching
- SWR — Lightweight, dari Vercel
- RTK Query — Bagian dari Redux Toolkit

Performance Optimization

Optimasi performa sangat penting untuk aplikasi production.

React Performance
// 1. React.memo - prevent unnecessary re-renders
const MemoizedComponent = React.memo(MyComponent);

// 2. useMemo - memoize expensive calculations
const expensiveValue = useMemo(() => {
  return heavyCalculation(data);
}, [data]);

// 3. useCallback - memoize functions
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]);

// 4. Code splitting with lazy loading
const Dashboard = lazy(() => import('./Dashboard'));

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <Dashboard />
    </Suspense>
  );
}

Testing Components

Testing memastikan component berfungsi dengan benar dan mencegah regresi.

Jest + React Testing Library
import { render, screen, fireEvent } from '@testing-library/react';

test('button increments counter', () => {
  render(<Counter />);
  
  // Find elements
  const button = screen.getByRole('button');
  const count = screen.getByText('Count: 0');
  
  // Interact
  fireEvent.click(button);
  
  // Assert
  expect(screen.getByText('Count: 1')).toBeInTheDocument();
});

Memilih Framework

Tidak ada framework terbaik untuk semua situasi. Pilih berdasarkan kebutuhan project dan tim.

React

  • Ekosistem terbesar
  • Fleksibel, library bukan framework
  • Job market terbesar
  • Cocok: SPA, team besar

Vue

  • Learning curve rendah
  • Documentation excellent
  • Semua dalam satu paket
  • Cocok: Startup, tim kecil

Svelte

  • No Virtual DOM
  • Bundle size kecil
  • Syntax paling simple
  • Cocok: Performance-critical

Angular

  • Full framework
  • Opinionated structure
  • TypeScript by default
  • Cocok: Enterprise apps
Meta Frameworks (Full-Stack):
- Next.js (React) — SSR, SSG, API routes
- Nuxt (Vue) — SSR, static generation
- SvelteKit (Svelte) — Full-stack Svelte
