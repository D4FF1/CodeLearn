Belajar JavaScript

Kuasai bahasa pemrograman web: sintaks modern, DOM, event, async, dan ES6+ features.

1

Pengenalan JS & Runtime di Browser

JavaScript adalah bahasa pemrograman yang membuat halaman web menjadi interaktif. Berbeda dengan HTML (struktur) dan CSS (style), JS memberikan "perilaku" atau logic. JS dijalankan oleh JavaScript Engine di browser (seperti V8 di Chrome atau SpiderMonkey di Firefox) yang menerjemahkan kode menjadi instruksi mesin.

JS
// Cara menampilkan output
console.log("Hello World!");  // ke console browser (F12)
alert("Pesan popup");          // dialog alert
document.getElementById("demo").innerHTML = "Teks berubah";  // manipulasi DOM
🔧 Cara menambahkan JS: External <script src="app.js"></script> atau internal <script>...</script> di akhir <body>.
2

Values & Data Types

Dalam JavaScript, setiap nilai memiliki tipe data. JS adalah bahasa dynamically typed, artinya Anda tidak perlu mendeklarasikan tipe data secara eksplisit. Tipe data dibagi menjadi Primitif (String, Number, Boolean, null, undefined, Symbol, BigInt) yang nilainya tidak bisa diubah (immutable), dan Object (termasuk Array dan Function) yang bisa menyimpan kumpulan data.

JS
let nama = "Budi";          // string: teks dalam tanda kutip
let umur = 25;               // number: integer atau float
let isActive = true;        // boolean: hanya true atau false
let kosong = null;           // null: nilai sengaja dikosongkan
let tidakTerdefinisi;         // undefined: variabel belum punya nilai
console.log(typeof nama);      // operator typeof mengecek tipe data
⚠️ Type coercion: JS otomatis mengubah tipe. "5" + 2 = "52" (string concat), tapi "5" - 2 = 3 (number conversion). Hati-hati bug akibat ini!
3

let/const/var (Variable Declaration)

Variabel adalah "wadah" untuk menyimpan data. Di JavaScript modern, ada tiga kata kunci untuk deklarasi: var (lama, function-scoped, bisa di-redeclare), let (modern, block-scoped, bisa diubah), dan const (modern, block-scoped, tidak bisa di-reassign). Gunakan const secara default, dan let jika nilai perlu diubah.

JS
// const: nilai tidak bisa di-reassign, tapi properti object/array bisa diubah
const PI = 3.14;
const arr = [1, 2, 3];
arr.push(4);  // ✅ diperbolehkan (mutasi isi array)

// let: block-scoped, nilai bisa diubah
let count = 0;
count = count + 1; // ✅ reassignment

// var: function-scoped, bisa bermasalah (hoisting)
var oldWay = "hindari di kode modern";
Best practice: Selalu gunakan const untuk variabel yang tidak berubah, gunakan let untuk loop atau counter. Hindari var untuk menghindari bug scope.
4

Operators: Aritmatika & Perbandingan

Operator digunakan untuk memanipulasi nilai. Aritmatika (+, -, *, /, %, **) untuk kalkulasi matematika. Perbandingan menghasilkan boolean (true/false). Sangat penting untuk membedakan == (loose equality, melakukan konversi tipe) dengan === (strict equality, membandingkan nilai DAN tipe).

JS
let a = 10, b = 3;
console.log(a + b, a - b, a * b, a / b, a % b, a ** b); // operasi matematika

// Perbandingan: ===, !==, >, <, >=, <=
console.log(5 == "5");   // true (loose, "5" diubah jadi 5)
console.log(5 === "5");  // false (strict, number vs string)
✅ Golden Rule: Selalu gunakan === dan !== untuk menghindari bug yang disebabkan oleh type coercion yang tidak terduga.
5

Truthy & Falsy

Dalam konteks boolean (misal kondisi if), JavaScript mengonversi nilai non-boolean menjadi true atau false. Nilai yang dianggap falsy (seperti false) ada 6: false, 0, "", null, undefined, NaN. Semua nilai lainnya dianggap truthy, termasuk string kosong "0", array kosong [], dan object kosong {}.

JS
if ("") console.log("Ini tidak akan tampil"); // "" adalah falsy
if ("hello") console.log("Ini tampil");     // string non-kosong adalah truthy
if (0) console.log("Tidak tampil");       // 0 adalah falsy
if ([]) console.log("Tampil");          // Array kosong adalah truthy!
💡 Tips: Gunakan !!value untuk mengonversi nilai menjadi boolean secara eksplisit, atau gunakan Boolean(value).
6

if/else Statement

Control flow memungkinkan kode mengambil keputusan. Statement if mengeksekusi blok kode jika kondisi bernilai truthy. else if untuk kondisi alternatif, dan else untuk fallback jika semua kondisi sebelumnya false. Untuk kondisi sederhana, gunakan Ternary Operator (kondisi ? nilaiTrue : nilaiFalse).

JS
let score = 85;
if (score >= 90) {
  console.log("Grade A");
} else if (score >= 80) {
  console.log("Grade B");
} else {
  console.log("Grade C");
}

// Ternary Operator (shorthand)
let status = score >= 70 ? "Lulus" : "Tidak Lulus";
7

switch Statement

Statement switch adalah alternatif dari if-else if ketika membandingkan satu nilai terhadap banyak kemungkinan (case). Lebih mudah dibaca untuk kondisi yang banyak. Penting: gunakan break di setiap case untuk mencegah "fall-through" (eksekusi lanjut ke case berikutnya).

JS
let day = "Monday";
switch (day) {
  case "Saturday":
  case "Sunday":
    console.log("Weekend!");
    break; // hentikan eksekusi di sini
  default:
    console.log("Weekday");
}
Tanpa break, kode akan terus mengeksekusi case berikutnya (fall-through). Ini kadang sengaja dilakukan untuk mengelompokkan beberapa case (seperti contoh di atas).
8

Loop: for, while, do-while

Loop digunakan untuk mengulang eksekusi kode. For loop ideal jika jumlah iterasi diketahui (init; condition; update). While loop cocok jika kondisi berhenti bergantung pada logika dinamis. Do-while minimal mengeksekusi kode 1 kali sebelum mengecek kondisi (post-test loop).

JS
// for: init (i=0); kondisi (i<5); update (i++)
for (let i = 0; i < 5; i++) { console.log("Iterasi ke-", i); }

// while: cek kondisi dulu, lalu jalankan
let j = 0;
while (j < 3) { console.log(j); j++; }

// do-while: jalankan dulu minimal 1x, lalu cek kondisi
let k = 0;
do { console.log(k); k++; } while (k < 0); // akan print 0 sekali
9

for...of & for...in

Loop modern untuk iterasi data. for...of mengiterasi nilai dari iterable (Array, String, Map, Set). for...in mengiterasi key/index dari Object. Gunakan for...of untuk array dan for...in untuk object.

JS
// for...of: iterasi nilai (array)
const fruits = ["🍎", "🍌", "🍊"];
for (const fruit of fruits) console.log(fruit); // 🍎, 🍌, 🍊

// for...in: iterasi key/property (object)
const user = { name: "Budi", age: 25 };
for (const key in user) {
  console.log(key, user[key]); // name Budi, age 25
}
10

Function Declaration vs Expression

Function adalah blok kode yang dapat digunakan ulang. Function Declaration menggunakan kata kunci function dan di-hoist (bisa dipanggil sebelum definisi). Function Expression menyimpan fungsi dalam variabel, tidak di-hoist. Expression lebih fleksibel (bisa passed as argument, anonymous).

JS
// Declaration - Hoisted (bisa dipanggil di atas definisi)
greet("Budi"); // ✅ Works
function greet(name) { return `Hello, ${name}!`; }

// Expression - Not Hoisted
const sayHi = function(name) { return `Hi, ${name}!`; };
sayHi("Andi"); // ✅ Harus setelah definisi
11

Arrow Functions

Arrow Function (diperkenalkan ES6) adalah sintaks singkat untuk menulis function expression. Menggunakan tanda =>. Perbedaan utama: Arrow function tidak memiliki this sendiri (mewarisi this dari scope luarnya), sehingga ideal untuk callback.

JS
// Syntax singkat
const add = (a, b) => a + b;      // implicit return
const double = n => n * 2;      // 1 param, tanpa kurung
const sayHello = () => console.log("Hi"); // no param

// Dengan block body (perlu return eksplisit)
const process = (value) => {
  const result = value * 2;
  return result;
};
Arrow function tidak cocok untuk method object (jika butuh this) atau constructor.
12

Parameters, Default, Rest Parameter

Parameters adalah variabel input fungsi. Default Parameter memberikan nilai fallback jika argumen tidak diberikan. Rest Parameter (...args) mengumpulkan sisa argumen menjadi array, sangat berguna untuk fungsi dengan jumlah argumen fleksibel.

JS
// Default parameter (ES6)
function greet(name = "Guest") { return `Hello ${name}`; }
greet();       // "Hello Guest"

// Rest parameter (...) mengumpulkan argumen jadi array
function sumAll(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sumAll(1, 2, 3, 4));  // 10
13

Scope & Hoisting

Scope menentukan aksesibilitas variabel. var adalah function-scoped (hanya terbatas di fungsi), sedangkan let dan const adalah block-scoped (terbatas di blok {}). Hoisting adalah perilaku JS memindahkan deklarasi ke atas scope saat kompilasi. var di-hoist dengan nilai undefined, sedangkan let/const menghasilkan error jika diakses sebelum deklarasi (Temporal Dead Zone).

JS
let globalVar = "global"; // Global scope
function test() {
  let localVar = "local"; // Function scope
  if (true) {
    let blockVar = "block"; // Block scope
  }
  // console.log(blockVar); // Error
}

// Hoisting dengan var
console.log(hoisted);  // undefined (bukan error)
var hoisted = "value";
14

Array: map, filter, reduce

Tiga method andalan untuk manipulasi array (Higher-Order Functions). map: mentransformasi setiap elemen menghasilkan array baru. filter: menyaring elemen berdasarkan kondisi. reduce: menggabungkan semua elemen menjadi satu nilai. Ketiganya tidak mengubah array asli (immutable).

JS
const numbers = [1, 2, 3, 4, 5];

// map: transformasi (kuadratkan semua)
const squared = numbers.map(n => n * n); // [1, 4, 9, 16, 25]

// filter: saring (ambil ganjil)
const odds = numbers.filter(n => n % 2 !== 0); // [1, 3, 5]

// reduce: akumulasi (jumlahkan semua)
const sum = numbers.reduce((acc, n) => acc + n, 0); // 15
⚡ Functional Programming: Chain method untuk operasi kompleks: numbers.filter(n => n > 2).map(n => n * 2).
15

Array: find, findIndex, some, every

Method pencarian dan validasi array. find: mencari elemen pertama yang cocok. findIndex: mencari index elemen pertama yang cocok. some: cek apakah ada minimal satu elemen cocok. every: cek apakah semua elemen cocok.

JS
const users = [{id:1, name:"Andi"}, {id:2, name:"Budi"}];

const user = users.find(u => u.id === 2);      // {id:2, name:"Budi"}
const index = users.findIndex(u => u.id === 2); // 1
const hasBudi = users.some(u => u.name === "Budi");  // true
const allHaveId = users.every(u => u.id !== undefined); // true
16

Destructuring (Array & Object)

Destructuring adalah cara mengekstrak nilai dari array atau properti dari object ke dalam variabel terpisah dengan sintaks ringkas. Sangat berguna untuk mengambil data dari API response atau config object.

JS
// Array destructuring (berdasarkan posisi)
const [a, b] = [10, 20]; // a=10, b=20

// Object destructuring (berdasarkan nama properti)
const { name, age } = { name: "Budi", age: 25 };

// Alias nama variabel
const { name: userName } = { name: "Budi" }; // userName = "Budi"

// Default value jika properti tidak ada
const { city = "Jakarta" } = { name: "Budi" };
17

Spread & Rest Operator (...)

Operator ... punya dua fungsi tergantung konteks. Spread: Memecah array/object menjadi elemen individual (saat assignment atau function call). Rest: Mengumpulkan elemen menjadi array (saat deklarasi parameter). Sangat powerful untuk cloning dan merging.

JS
// Spread (memecah)
const arr1 = [1, 2], arr2 = [3, 4];
const combined = [...arr1, ...arr2];   // [1,2,3,4]
const newObj = { ...{a:1}, b:2 };     // {a:1, b:2}

// Rest (mengumpulkan)
const [first, ...rest] = [10, 20, 30];  // first=10, rest=[20,30]
const { a, ...others } = { a: 1, b: 2, c: 3 }; // others={b:2,c:3}
18

Objects (Properti & Method)

Object adalah tipe data kompleks untuk menyimpan pasangan key-value. Key biasanya string, value bisa berupa tipe data apa saja termasuk fungsi (disebut method). Akses properti bisa dengan dot notation (obj.key) atau bracket notation (obj["key"]) untuk key dinamis.

JS
const car = {
  brand: "Toyota",  // property
  year: 2022,
  start() {       // method (shorthand ES6)
    console.log("Engine started");
  }
};

car.brand;        // "Toyota" (dot notation)
car["year"];        // 2022 (bracket notation)
car.start();       // memanggil method
19

DOM: querySelector & querySelectorAll

DOM (Document Object Model) adalah representasi HTML sebagai pohon objek yang bisa dimanipulasi JS. querySelector mengembalikan elemen pertama yang cocok dengan CSS selector. querySelectorAll mengembalikan NodeList semua elemen yang cocok. Gunakan ini untuk memilih elemen sebelum manipulasi.

JS
// Memilih elemen (gunakan CSS selector)
const title = document.querySelector("h1");       // elemen 
 pertama
const container = document.getElementById("app"); // by ID (lebih cepat)
const buttons = document.querySelectorAll(".btn"); // NodeList

// Mengubah konten
title.textContent = "Judul Baru";    // teks saja
title.innerHTML = "Judul Baru"; // interpret HTML
20

DOM: createElement, appendChild, remove

JS bisa membuat, menambah, dan menghapus elemen HTML secara dinamis. createElement membuat node baru. appendChild menambahkan node sebagai child terakhir. remove menghapus node dari DOM. Operasi ini memungkinkan halaman web berubah tanpa reload.

JS
// Membuat elemen baru
const newDiv = document.createElement("div");
newDiv.textContent = "Hello World";
newDiv.classList.add("box");

// Menambahkan ke DOM
document.body.appendChild(newDiv);

// Menghapus elemen
const old = document.getElementById("removeMe");
old.remove(); // hapus dari DOM
21

DOM: Event Listeners

Interaktivitas web dibangun di atas Events (klik, hover, scroll, dll). addEventListener adalah cara standar untuk mendaftarkan fungsi (callback) yang akan dijalankan saat event terjadi. Fungsi ini menerima objek event yang berisi info tentang kejadian tersebut.

JS
const btn = document.querySelector("#myButton");

// Tambah event listener
btn.addEventListener("click", () => {
  alert("Button diklik!");
});

// Event object untuk detail
btn.addEventListener("click", (event) => {
  console.log(event.target); // elemen yang diklik
  console.log(event.type);   // "click"
});
22

Event Types: click, input, change, submit

Ada puluhan event types. Yang sering digunakan: click (klik mouse), input (perubahan nilai real-time), change (perubahan nilai setelah blur/commit), submit (form disubmit). Gunakan event.preventDefault() untuk mencegah perilaku default (misal refresh saat submit form).

JS
const input = document.querySelector("#username");

// input: trigger setiap ketikan
input.addEventListener("input", (e) => {
  console.log("Nilai saat ini:", e.target.value);
});

// change: trigger setelah blur (user selesai edit)
input.addEventListener("change", (e) => {
  console.log("Final value:", e.target.value);
});

// submit: tangkap form submission
const form = document.querySelector("form");
form.addEventListener("submit", (e) => {
  e.preventDefault();  // cegah refresh halaman
  console.log("Form submitted!");
});
23

Form Handling (mengambil nilai)

FormData API menyederhanakan pengambilan nilai form. Metode tradisional: akses input.value satu per satu. Metode modern: gunakan new FormData(form) untuk mengambil semua nilai sekaligus dalam format key-value pair.

JS
const form = document.getElementById("loginForm");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  
  // Cara modern dengan FormData
  const formData = new FormData(form);
  const username = formData.get("username"); // ambil by name
  const password = formData.get("password");
  
  console.log({ username, password });
});
24

Fetch API & Async/Await

Fetch API digunakan untuk mengirim/menerima data dari server (HTTP Request) tanpa reload halaman (AJAX). Fetch mengembalikan Promise. async/await adalah sintaks modern untuk menangani Promise agar kode lebih mudah dibaca seperti synchronous.

JS
async function fetchUser() {
  try {
    // await menunggu response sampai selesai
    const response = await fetch("https://jsonplaceholder.typicode.com/users/1");
    
    const user = await response.json(); // parse JSON
    console.log(user);
  } catch (error) {
    console.error("Gagal fetch:", error);
  }
}
fetchUser();
Promise: Objek yang mewakili nilai yang belum tersedia (tapi akan tersedia nanti). await hanya bisa digunakan di dalam fungsi async.
25

Error Handling: try/catch/finally

Error handling mencegah aplikasi crash saat terjadi kesalahan. Blok try berisi kode yang mungkin error. catch menangkap error jika terjadi. finally selalu dijalankan baik error maupun tidak (cocok untuk cleanup).

JS
try {
  // Kode yang berisiko error
  let result = riskyOperation();
  console.log(result);
} catch (error) {
  // Handler jika error
  console.error("Terjadi error:", error.message);
} finally {
  // Selalu dieksekusi (cleanup)
  console.log("Proses selesai.");
}
26

localStorage & sessionStorage

Web Storage API menyimpan data di browser. localStorage: data tetap ada meski browser ditutup (persistent). sessionStorage: data hilang saat tab ditutup. Keduanya menyimpan data dalam format string (key-value).

JS
// localStorage (persistent)
localStorage.setItem("theme", "dark");    // simpan
const theme = localStorage.getItem("theme"); // ambil
localStorage.removeItem("theme");           // hapus

// sessionStorage (hilang saat tab ditutup)
sessionStorage.setItem("temp", "data sementara");
Storage hanya bisa menyimpan string. Untuk menyimpan object/array, gunakan JSON.stringify() saat simpan dan JSON.parse() saat ambil.
27

JSON.stringify & JSON.parse

JSON (JavaScript Object Notation) adalah format pertukaran data ringan. JSON.stringify() mengubah object JS menjadi string JSON (untuk kirim ke server atau simpan di storage). JSON.parse() mengubah string JSON kembali menjadi object JS.

JS
const user = { name: "Budi", age: 25 };

// Object -> String JSON
const jsonString = JSON.stringify(user);
console.log(jsonString);  // '{"name":"Budi","age":25}'

// String JSON -> Object
const parsedUser = JSON.parse(jsonString);

// Untuk storage:
localStorage.setItem("user", JSON.stringify(user));
const savedUser = JSON.parse(localStorage.getItem("user"));
28

Modules: import / export

ES6 Modules memungkinkan kode dibagi ke file terpisah untuk modularitas. export mengekspos fungsi/variabel dari file. import mengimpor dari file lain. Ini adalah standar modern untuk manajemen kode di proyek besar.

JS
// === utils.js ===
export function greet(name) { return `Hello ${name}`; }
export const VERSION = "1.0";
export default function main() {} // default export

// === main.js ===
import main, { greet, VERSION } from "./utils.js";
greet("Budi"); // "Hello Budi"
Untuk menggunakan modules di browser, tambahkan <script type="module" src="main.js"></script> di HTML.

Practice: Calculator

Calculator melatih manajemen state (nilai display, operator, memory). Logika utama: append angka ke string display, simpan operator, lalu hitung saat tombol "=" ditekan. Gunakan eval() untuk demo sederhana (hati-hati di production).

Practice: Modal Popup dengan JS

Modal adalah komponen UI overlay. Kontrol visibility dengan CSS display atau class toggle. Penting untuk menambahkan event listener pada backdrop (background gelap) agar modal tertutup saat user klik di luar konten modal.

JS
const modal = document.getElementById("myModal");
const openBtn = document.getElementById("openModalBtn");

function openModal() { modal.style.display = "flex"; }
function closeModal() { modal.style.display = "none"; }

openBtn.addEventListener("click", openModal);
closeBtn.addEventListener("click", closeModal);

// Klik di luar modal (backdrop) untuk tutup
window.addEventListener("click", (e) => {

});

Practice: Image Gallery Controller

Gallery/Slider membutuhkan tracking index aktif. Fungsi next() increment index (dengan wrap-around ke 0 jika di akhir). Fungsi prev() decrement (dengan wrap-around ke akhir jika di 0). Tampilkan hanya gambar dengan index yang cocok.

JS
let currentIndex = 0;
const images = document.querySelectorAll(".gallery img");

function showImage(index) {
  images.forEach((img, i) => {
    img.style.display = i === index ? "block" : "none";
  });
}

function next() {
  currentIndex = (currentIndex + 1) % images.length; // 0,1,2,3,0,1...
  showImage(currentIndex);
}

function prev() {
  // +length untuk handle negative index
  currentIndex = (currentIndex - 1 + images.length) % images.length;
  showImage(currentIndex);
}
showImage(0); // init

Practice: Stopwatch

Stopwatch menggunakan setInterval untuk update waktu berkala. Variabel intervalId disimpan untuk bisa dihentikan dengan clearInterval. Format waktu dari milidetik ke menit:detik:milidetik.

JS
let time = 0, interval;

function startTimer() {
  if (interval) return; // cegat multiple interval
  interval = setInterval(() => {
    time += 10; // increment 10ms
    updateDisplay();
  }, 10); // update setiap 10ms
}

function stopTimer() { 
  clearInterval(interval); 
  interval = null; 
}

function resetTimer() { 
  stopTimer(); 
  time = 0;

Best Practices & Code Organization

Menulis kode yang "clean" sama pentingnya dengan kode yang berfungsi. Prinsip utama: gunakan nama variabel deskriptif, hindari "magic numbers", pecah kode besar jadi fungsi kecil (Single Responsibility), dan selalu handle error. Kode yang bersih mudah dibaca, di-debug, dan dikembangkan.

JS
// ❌ Bad: Magic number, nama tidak jelas
const x = 86400;
if (d > x) { ... }

// ✅ Good: Konstanta bernama, jelas maknanya
const SECONDS_IN_DAY = 86400;
if (duration > SECONDS_IN_DAY) { ... }

// ✅ Early Return Pattern
function processUser(user) {
  if (!user) return null; // keluar awal jika invalid
  
  // logic utama di sini tanpa nested if
  return user.name;
}

// ✅ Async/Await untuk kode asinkron yang bersih
async function getData() {
  try { ... } catch (e) { ... }
}
Baca "Clean Code JavaScript" untuk panduan lebih lanjut tentang penulisan kode yang profesional.
